Desktop Wireframes

When I began creating this website, I quickly realized that my original wireframes didn’t function properly within standard web design rules and conventions. As I developed the site, I found that the initial layout lacked the structure needed for intuitive navigation and accessibility. For example, originally the navigation bar didn't have links in order of importance, they were randomally placed, breaking certain conventions which can confuse the user. I also restructured the content areas to follow a more traditional F-pattern layout, which aligns better with how users typically scan websites. Along the way, I decided to change the entire aesthetic of the website: instead of a clean, minimalist look, I adopted a dark, vintage theme that better highlights the statues and artwork featured on the site. To fit this new artistic vision, I introduced creative elements such as an image background, serif typography, and warmer tones, all of which give the site a more immersive and curated feel. Additionally, I refined the layout to improve readability, spacing out content sections more thoughtfully and using contrast to guide the user's eye, ensuring a smoother and more engaging browsing experience.

Mobile Wireframes

Annotations

Navigation Bar

The navigation bar plays a key role in the usability of the website by guiding users through its various sections. I organized the links from left to right based on their importance, ensuring that this order remained consistent across all pages. Functionally, it offers clear and direct access to the main areas of the site, making navigation straightforward. Maintaining a consistent navigation structure enhances the user experience by making information easier to locate.

Main Layout

Each page of the website uses a vertical scroll layout, allowing users to seamlessly view all content by simply scrolling down. To enhance usability, a "Back to Top" button is included, enabling users to quickly return to the top of the page without manually scrolling. This feature improves navigation efficiency, especially on content-rich pages.

Responsive Design

The website is designed to adapt to various screen sizes, ensuring it remains functional and visually appealing across different devices. This responsive design enhances the user experience by providing seamless access, whether users are browsing on a desktop or smartphone.

BreadCrumb Navigation

This secondary navigation allows users to navigate between all the blogs without having to go back to the main blog page. This can reduce frustration within users, as they won't have to go back to find a new blog. This is especially useful for mobile users, as they can easily navigate between blogs without having to scroll back to the top of the page to click the previous and next buttons.

Buttons

There is previous and next buttons, which allows users to navigate between blog. There are buttons for the wireframes, style process and IXD process, so that users can easily navigate the design section. These buttons are designed to be easily clickable, even on mobile devices, ensuring a smooth user experience.

Reflection

My wireframes have evolved significantly since I first created them. Initially, they helped guide the structure and layout of my website, but once I began implementing CSS and seeing the design come to life in the browser, I realised that some of my original ideas weren’t as functional or visually effective as I had imagined.

One of the main changes I made was to the layout of my blog page. Originally, I had positioned the secondary navigation bar on the side of the page to separate it from the main navigation. However, once I started developing it with CSS, I found that this layout felt disconnected and inconsistent with the rest of the site. To improve the overall flow and cohesion of the design, I moved the secondary navigation to the top of the page, directly beneath the main navigation bar.

This adjustment created a more seamless and intuitive user experience, as it mirrored the structure of the main site navigation. It also helped improve visual alignment, especially on smaller screens, and made the layout more responsive. Changing the wireframes during development allowed me to adapt the design to better suit the practical needs of the user and ensure consistency across the entire website.
